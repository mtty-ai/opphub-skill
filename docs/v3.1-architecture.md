### 1.2 login 现状 (死结)

```bash
const dc = await getDeviceCode();
openBrowser(dc.verification_uri_complete);   # 硬编码 open
const tok = await pollToken(dc.device_code); # 阻塞轮询 600s
saveToken(tok);
```

**问题**: bot 跑 = 没 stdin / 不能 spawn `open` / 不能阻塞 600s = **chat 里跑不了**

### 1.3 configure 现状 (真空)

- skill **完全没有** configure 命令
- plugin 跑 `openclaw opphub configure` = **inquirer terminal** 单选
- 用户 SSH / 移动端 / 没开 terminal = **配不了**

### 1.4 推送链路 (INTERNAL.md §4 钉死)

```
opphub-server (撮合触发)
  ↓ pushPendingMessage() 写 DB
ws-server.js pullAndPush 5s 扫
  ↓ WS 推 event
opphub-plugin (本机常驻 · channel plugin)  ← 必须 plugin 在
  ↓ runtime.channel.inbound.run(...)
OpenClaw runtime → 按 user 已配 IM channel
  ↓
飞书 / 微信 / 钉钉 → 用户手机
```

**死结**:
- 没 plugin = ws-client 不存在 = server 推过来**静默丢失**
- configure 完不引导装 plugin = **配置是空的**

## 2. 设计原则 (钉死)

| 原则 | 做法 |
|---|---|
| bot 不能开浏览器 | device flow 拆 2 步: **发 URL → 用户回 code** |
| bot 不能推 inquirer | 用 **飞书 interactive card** (dropdown / button) |
| bot 不能跨 user 推 | configure 只调 server PATCH, **不传 peer** |
| 同 Keychain 凭证 | skill 跟 plugin 写**同一份 Keychain** (`openclaw-opphub-uat` / `opphub:default`) |
| install-headless 路径保留 | plugin CLI `opphub configure` / `opphub login` **保留** (SSH / installer) |
| configure 完必须引导装 plugin | 5 步闭环 (见 §4) |
| 强引导不强制 | "暂时跳过" 按钮允许跳过, 不劝退 |

### Step 5 · 收尾

- 用户点 "暂时跳过" (skip_install): 不强推, 但下次 @bot 偶合状态 还提醒一次
- 用户点 "我已装好, 验证" (verify_link): skill 调 `openclaw opphub status` + `openclaw cron list` 看链路
- 用户点 "一键装 plugin" (install_plugin): bot **不真调** `openclaw plugins install`(SSO SSH 场景跑不了),**只发文案 + 安装链接**,让用户本地 terminal 跑



> **历史脉络**:
> - **7/15 web P0 规划**: 蒸馏设计 — `OpcProfile.distilledJson` + `embedding vector(768)` + `lib/distill.ts` LLM 填空字段

**与 7/15 蒸馏设计的本质区别**:

|---|---|---|
| 录入 | 表字段 `mainSkill/subSkills/experience/...` + LLM 填空 | **随便说一段 / 贴 URL / 上传 PDF, 不强制字段** |
| 存储 | `OpcProfile.distilledJson` + `embedding` (表还在) | **整段 chunks + embedding + tsvector 全文索引** |
| OpcProfile | 保留 mainSkill/subSkills/experience 字段 | **整段 rawText + embedding, 字段弱化/砍** |
| OpcSkillCard | 多张字段卡 `title/category/price/deliverDays/portfolio` | **多 chunks, 每 chunk 独立向量** |
| 检索 | embedding 相似度 + tag 过滤 | **pgvector HNSW + tsvector 全文检索, 不 JOIN** |
| KnowledgeTag | 必填 | **保留** (蒸馏产物, 可空, 可自举) |

**死结 (不填 = 装了也白装)**:
- plugin 装好 → server 能推 → **但 server 不知道用户能干啥 = 推送不精准**
- 没知识库条目 → matching 无依据 → 推过来商机质量差
- **录入知识库 = 让撮合匹配有依据**

#### 6.2 手动录入 (开放式, 不限字段) · channel-agnostic intent

**Skill 调用**:

```js

await bot.askFreeText({

  prompt: "🟡 告诉我你能干啥 (随便说, 没字段)",

  actions: [
    { id: "upload_file",   label: "📎 上传文件 (BP PDF)", style: "primary" },
    { id: "paste_url",     label: "🔗 贴个人主页 URL",    style: "default" },
    { id: "auto_fill",     label: "skill 帮我自动拼",     style: "default",
      payload: { action: "knowledge-autofill" } },
    { id: "skip",          label: "跳过",                 style: "danger"  },
  ],

```

**Runtime 转译**:

| channel | free-text + 上传 + 链接 渲染 |
|---|---|
| 飞书 DM | 文本 + 输入框 + actions (上传按钮 + URL + 自动拼 + 跳过) |
| 微信 iLink | 文本 + emoji + 输入引导 (后上上传走附件 API) + 编号选择 |
| Slack | Block Kit · input block + actions (button) + link button |
| Discord | Embed + 输入提示 + components (buttons) + URL button |
| Telegram | InlineKeyboard + 等待下一条消息文本 |
| CLI | inquirer.input (info hint) + 等待 stdin |
| web | HTML textarea + file input + form submit |

**用户回复处理**:

```js
const userInput = await bot.readNext();
switch (userInput.intent) {
  case "free_text":
    // 飞书 / Slack / web textarea / CLI 等
    await ingestKnowledge({ rawText: userInput.text, sourceType: "manual" });
    break;
  case "file_upload":
    // 飞书 / Slack attachments / 微信附件 / web file input
    await ingestKnowledge({ file: userInput.file, sourceType: "upload" });
    break;
  case "url_paste":
    await ingestKnowledge({ url: userInput.url, sourceType: "url" });
    break;
  case "auto_fill":
    // 调用 bin/opphub-knowledge-autofill (15 源)
    await autoFillKnowledge();
    break;
  case "skip":
    /* 跳过 */
    break;
}
```

**server 端入库管道** (`/api/knowledge/ingest` POST):

- 切片 (chunker, 默认 512 tokens / 50 overlap)
- 算 embedding (BGE-M3, 768 维)
- 蒸馏 tags (LLM 抽 3-10 个, 可空)
- 存 `OpcKnowledgeEntry` 表
- 写 `tsvector` 全文索引

#### 6.3 自动抓取 (无感, 首选)

**数据源** (skill 自动从这些地方抓, 拼成**一段原文**):

**A. 本机状态** (6 源)

|---|---|---|
| Keychain token | 已登录, 拿 opcId | `lib/keychain.js` 读 access_token |

| plugin `outbox.log` | 历史推送记录 | 读 `~/.opphub-plugin/outbox.log` |



| 来源 | 抓什么 | 怎么抓 |
|---|---|---|
| `~/.openclaw/workspace-dev/SOUL.md` | bot 身份 + 红线 + 写作风格 | skill turn 读 / 装 skill 时 copy |
| `~/.openclaw/workspace-dev/USER.md` | 用户名 / OpenID / 公司 / 上下文 | skill turn 读 |

**为什么 memory 是最真实的画像**:
- `MEMORY.md` 红线 → 用户的判断风格 / 拍过的板
- `memory/2026-07-XX.md` → 用户**当天**在做什么项目、跟谁合作
- `wiki/main/syntheses/` → 累积的项目决策 / 架构 / 关键对话
- 这些是 **LLM 猜不到**的 — 只有用户本机有


| 来源 | 抓什么 | 怎么抓 |
|---|---|---|
| `minimax__web_search` | 公司名 / 业务 / 案例 | skill turn 调 |
| `web_fetch` | 拉 URL(知乎 / 小红书 / 公众号 / 官网) | skill turn 调 |

**问题**: 6.3 自动抓取的 6 源**只反映本机状态**(公司名、装的 plugin、推送历史),**反映不出"用户业务能干啥"**。要拿到业务/能力/案例,必须**主动调 LLM 工具**。

**OpenClaw runtime 提供给 skill 的工具**:
- `minimax__web_search` — 联网搜索 (可搜公司名 / 创始人 / 产品)
- `web_fetch` — 拉 URL 内容 (个人主页 / 知乎 / 小红书 / 公众号 / 官网)
- `minimax__understand_image` — 解析图片 (用户发作品截图 / 名片 / 介绍图)
- `pdf` — 解析 PDF (用户上传 BP / 案例集 / 团队介绍)
- LLM 自身 (skill turn 里的 bot) — 文本推理 / 摘要

**姿势 A: skill 主动搜 + 推理** (skill turn 内):

```
用户 @bot "偶合配置"
  ↓
skill turn 跑
  ↓
(1) 读 userinfo 拿 companyName + legalPerson
  ↓
(2) skill 调 minimax__web_search(query="上海睿驰嘉禾数字传媒 公司 业务", count=5)
  ↓
(3) skill 调 web_fetch(命中 URL) → 拉详情
  ↓
(4) LLM 汇总 → 推断:
    - 主营业务: AI OPC 撮合平台 / 偶合
    - 核心能力: 多通道 IM 接入 / OpenClaw plugin 开发 / 飞书集成
    - 团队规模: 5-10 人 (估)
    - 案例: 已上线 opphub-plugin / opphub-server / opphub-skill 3 个产品
  ↓
(5) 拼成 rawText → POST /api/knowledge/ingest { sourceType: "auto-llm" }
```

## 7. 关键设计点

| 点 | 设计 |
|---|---|
| **配置生效不依赖 plugin** | PATCH server 立刻生效; plugin 装好 ws-client 自动连 (读 Keychain 同 opcId) |
| **没装 plugin 不阻塞** | configure 也能跑 (只 PATCH + cron); 只是引导说"不装就收不到" |
| **强引导不强制** | "暂时跳过" 按钮允许用户跳过 (避免劝退); 后续 @bot 偶合状态 还提醒一次 |
| **同 Keychain** | skill 跟 plugin 读同一份 Keychain, **plugin 装好自动接** |
| **闭环覆盖 5 状态** | `missing` → `valid` → `configured` → `plugin_installed` → `link_verified` (@bot 偶合状态 全显示) |
| **install 按钮不真调** | bot 跑 `openclaw plugins install` = 失败 (没 TTY); **按钮只发文案, 让用户本地 terminal 跑** |
| **verify 链路** | 调 `openclaw opphub status` + `openclaw cron list` 看 plugin / cron / ws 三件套 |

## 11. 风险 + 缓解

| 风险 | 缓解 |
|---|---|
| bot 调 login 后, 用户不点同意怎么办 | 5 分钟后 device_code 过期, bot 提示 "已超时, 再 @bot 偶合登录" |
| 同 Keychain 多写冲突 (skill 写 / plugin 写) | 都调同一个 `lib/keychain.js`, 文件锁 + 写后 fsync, plugin v0.4.0 已实现 |
| configure 提交后, server 推就用新通道 | server push 走 `server-side selected` (`ws-server.js` 查 `OpcChannel.selected` 推), **不读 plugin 缓存**, 所以配置即时生效 |
| install 按钮误以为能装 | 按钮不发 install 命令, 只发文案, 引导用户本地 terminal 跑 |
| 没装 plugin 用户点 "暂时跳过" | 不阻塞, 但下次 @bot 偶合状态 还提醒一次 (2 次后改 7 天一次提醒) |
